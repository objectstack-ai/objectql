# ObjectQL Trademark Usage Guidelines

## Trademark Policy

The "ObjectQL" name and logo are trademarks of the ObjectQL project. This policy outlines how you can use these trademarks.

## ‚úÖ Permitted Uses

You **MAY** use the ObjectQL name and logo:

1. **To refer to the ObjectQL project itself**
   - "This application uses ObjectQL as its data layer"
   - "Built with ObjectQL"
   - "Powered by ObjectQL"

2. **In technical documentation and tutorials**
   - Blog posts, articles, and tutorials about ObjectQL
   - Books and educational materials
   - Conference presentations and talks

3. **For community activities**
   - User groups and meetups (e.g., "ObjectQL Users Meetup")
   - Non-commercial training events
   - Community forums and discussion groups

4. **In open-source projects**
   - Projects that integrate with ObjectQL
   - Extensions and plugins for ObjectQL
   - You must make it clear that your project is not officially endorsed

## ‚ùå Prohibited Uses

You **MAY NOT** use the ObjectQL name or logo:

1. **In a way that suggests official endorsement**
   - Do not imply your product is officially part of ObjectQL
   - Do not use "ObjectQL" as the primary name of your product
   - Examples of prohibited names:
     - "ObjectQL Cloud" (unless officially licensed)
     - "ObjectQL Pro"
     - "ObjectQL Enterprise Edition"

2. **For competing products or services**
   - Do not use ObjectQL trademarks to market a competing database or ORM
   - Do not create a fork and call it "ObjectQL"
   - If you fork the code, you must rename it

3. **In domain names (without permission)**
   - Do not register domains like "objectql.com", "objectql.io", etc.
   - Community resources should use descriptive names like "objectql-community.org"

4. **In a misleading or harmful way**
   - Do not use trademarks in a way that damages our reputation
   - Do not associate ObjectQL with illegal or unethical activities

## üè¢ Commercial Use

If you want to use ObjectQL trademarks for commercial purposes:

1. **SaaS providers** offering ObjectQL-based services should:
   - Use descriptive names like "Database Platform powered by ObjectQL"
   - Obtain a trademark license for prominent use
   - Consider a commercial license (see LICENSE)

2. **Product companies** integrating ObjectQL:
   - May mention ObjectQL in feature lists and marketing materials
   - Should use phrases like "Compatible with ObjectQL" or "Integrates with ObjectQL"
   - For prominent logo placement, please request permission

3. **Training and consulting** services:
   - May advertise ObjectQL expertise
   - Should clarify you are not officially endorsed unless you are
   - Company names like "Acme ObjectQL Consulting" require prior approval

## üìù How to Request Permission

For uses not clearly permitted above, please contact:
- Email: trademarks@objectql.org
- Include:
  - Description of intended use
  - Mockups or examples if applicable
  - Your relationship to the ObjectQL project

We aim to respond within 2 weeks.
